Student Python Model Summary

Created as part of the GEOG5990: Programming for Geographical Information Analysts: Core Skills, a module incorporated into the MSc Geographical Information Systems at the University of Leeds.

All model code is located within the Assessment 2 Repository, along with the associated files for the README, GNU General Public License v3.0 Licensing and Documentation. The README contains a Model Summary, which should be read after this page, followed by a detailed Model Guidance PDF before running the Model. The Model Guidance contains detail on the software intention, current issues, design, Unified Modelling Language (UML) Diagrams and future developments.

  • This animated agent-based model uses Spyder (Anaconda 3) Python 3.7.
  • The model code interacts with an agent framework code and an environment to produce coordinates on a 300 by 300 'raster' grid. Both Drunks and Police xy coordinates are produced randomly. Code within the 'move, check_drunks and check_police' functions enable Drunks to move away from Police and likewise Police closer to Drunks. The number of Drunks is already established as 25, so this a home for each Drunk. The number of Police is determined by a Graphical User Interface (GUI) slider, between 1 to 5 Police can be chosen.
  • Both the ‘Drunks’ and ‘Police’ variables are defined as ‘Agents’ at distinctly defined separate times within the model. The Drunks attempt to- move, check for Police, produce a trail showing agent movements in real time and currently unsuccesfully get home and stop once they reach home.
  • Police attempt to - move, check for Drunks and guide Drunks home.
  • Once all Drunks reach home, or Police reach coordinates (300, 300) or the maximum Number of Iterations is met, or the code is manually closed via the 'Step E' button/through the menu bar, the model stops. The first two model stop reasons are not currently successful despite multiple attempts.
  • The Model can be rerun by closing and reopening the Model.
  • The code uses core Python language with agent based modelling, object-oriented approach and continuous integration.
